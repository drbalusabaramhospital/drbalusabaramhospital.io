# Firestore Data Schema Reference

Sabaram Hospital Patient Queue Management System  
Firebase Project: `sabaramhospital-67090`

---

## Collections at a Glance

| Collection | Created by | Public read | Public create | Auth required for update/delete |
|---|---|---|---|---|
| `doctors` | Admin UI | Yes | No | Yes |
| `users` | Manual (Firebase console) | No | No | Yes |
| `visits` | Reception / Self-service kiosk | Yes | Yes | Yes |
| `patients` | Reception only | No | No | Yes |
| `clinicSettings` | Admin UI | Yes | No | Yes |
| `counters` | Auto on first use | Yes | Yes (full write) | — |
| `backups` | Backup page | No | No | Yes |

> **Note:** `visits` allows public `create` and `counters` allows full public write so the self-service kiosk can book tokens without login. `patients` is staff-only — the kiosk stores patient details on the visit document instead.

## Collection: `doctors`

**Purpose:** Stores each doctor's profile, consultation settings, and current availability status.

**Document ID:** Auto-generated by Firestore (via Admin UI).

**Relationships:** `users.doctorId` must match a document ID in this collection.

### Fields

| Field | Type | Required | Description | Example |
|---|---|---|---|---|
| `name` | string | Yes | Doctor's full name with title | `"Dr. Balu Sabaram"` |
| `specialization` | string | Yes | Medical specialization shown on queue board | `"General Medicine"` |
| `consultationFee` | number | Yes | Fee in Indian Rupees | `300` |
| `seedActMinutes` | number | Yes | Average consultation time in minutes, used to estimate wait times | `8` |
| `room` | string | Yes | Room name shown on queue board and voice announcements | `"Room 1"` |
| `status` | string | Yes | `"active"` or `"inactive"`. Only active doctors appear in reception dropdown | `"active"` |
| `onBreakUntil` | number \| null | Yes | Epoch milliseconds when the break ends. `null` when not on break | `null` |

### Example Document

```json
{
  "name": "Dr. Balu Sabaram",
  "specialization": "General Medicine",
  "consultationFee": 300,
  "seedActMinutes": 8,
  "room": "Room 1",
  "status": "active",
  "onBreakUntil": null
}
```

## Collection: `users`

**Purpose:** Maps each Firebase Auth UID to a role and optional doctor profile link. Controls which page the user is redirected to after sign-in.

**Document ID:** The Firebase Auth UID of the user (copied from Firebase console → Authentication).

**Relationships:** If `role` is `"doctor"`, the `doctorId` field must match a document ID in the `doctors` collection.

### Fields

| Field | Type | Required | Description | Example |
|---|---|---|---|---|
| `displayName` | string | Yes | Human-readable name shown in the app | `"Dr. Balu Sabaram"` |
| `role` | string | Yes | One of: `"admin"`, `"reception"`, `"doctor"`. Determines redirect on sign-in | `"doctor"` |
| `doctorId` | string | Only if `role = "doctor"` | Document ID in `doctors` collection. Must match exactly | `"abc123xyz"` |
| `email` | string | No | For reference only; not used by the app | `"doctor@sabaram.com"` |

### Role Redirects

| Role | Redirected to |
|---|---|
| `admin` | `admin.html` |
| `doctor` | `doctor.html` |
| `reception` (or anything else) | `reception.html` |

### Example Document

```json
{
  "displayName": "Dr. Balu Sabaram",
  "role": "doctor",
  "doctorId": "abc123xyz",
  "email": "doctor@sabaram.com"
}
```

> **Note:** This collection is never written by the app UI — all `users` documents must be created manually in the Firebase console by an admin.

## Collection: `clinicSettings`

**Purpose:** Global configuration for the clinic — name, UPI, prescription quality, voice announcements.

**Document ID:** Always `"main"` (single document).

### Fields

| Field | Type | Required | Description | Example |
|---|---|---|---|---|
| `clinicName` | string | Yes | Shown on printouts and the queue board | `"Sabaram Hospital"` |
| `selfServiceUrl` | string | Yes | Full URL of `self-service.html`; encoded into patient card QR codes | `"https://drbalusabaramhospital.github.io/clinic/self-service.html"` |
| `upiVpa` | string | No | UPI Virtual Payment Address for fee collection | `"sabaram@upi"` |
| `upiMerchantName` | string | No | Merchant name shown in patient's UPI app | `"Sabaram Hospital"` |
| `upiMerchantCode` | string | No | Optional merchant category code from bank | `"5099"` |
| `prescriptionQuality` | number | No | JPEG quality for prescription photos, 0.0–1.0 | `0.6` |
| `prescriptionMaxPx` | number | No | Maximum pixel dimension when capturing prescription images | `1200` |
| `visitHistoryCount` | number | No | Number of past visits shown in doctor's consultation view | `3` |
| `voiceEnabled` | boolean | No | Whether voice announcements are active | `true` |
| `voiceRepeat` | number | No | How many times each announcement is spoken (1, 2, or 3) | `2` |
| `voiceInterval` | number | No | Seconds between repeat announcements. `0` disables auto-repeat | `5` |
| `voiceLanguages` | array | No | Language configurations for announcements (see below) | see below |

### `voiceLanguages` Array Item Structure

```json
{
  "lang": "en-IN",
  "template": "Token number {token}, please proceed to {room}",
  "enabled": true
}
```

| Sub-field | Type | Description |
|---|---|---|
| `lang` | string | BCP-47 language tag (e.g. `"en-IN"`, `"ta-IN"`, `"hi-IN"`) |
| `template` | string | Announcement text. Supported placeholders: `{token}`, `{room}`, `{name}`, `{doctor}` |
| `enabled` | boolean | Whether this language is announced |

## Collection: `visits`

**Purpose:** One document per patient visit per day. Tracks the entire lifecycle of a patient's visit: token issuance, queue position, consultation, vitals, fees, and prescriptions.

**Document ID:** Auto-generated by Firestore.

**Relationships:**
- `patientId` references a document in `patients` — **may be `null`** for visits created via the self-service kiosk (patient details are stored directly on the visit; reception links the patient record when processing the token)
- `doctorId` references a document in `doctors`

> **Self-service kiosk note:** When a patient books via the QR code kiosk (`self-service.html`), the visit is created without authentication. `patientId` and `patientCode` will be `null`. `patientMobile` and `patientAddress` are stored directly on the visit document. Reception should search for the patient by mobile when they arrive and link the record manually.
- `preferredDoctorIds` is an array of document IDs from `doctors`

### Fields

| Field | Type | Required | Description | Example |
|---|---|---|---|---|
| `patientId` | string | Yes | Document ID in `patients` collection | `"pABC123"` |
| `patientCode` | string | Yes | Denormalized patient code for display | `"CLN-000001"` |
| `patientName` | string | Yes | Denormalized patient name for display | `"Ravi Kumar"` |
| `patientAge` | number | Yes | Age at time of visit | `45` |
| `patientGender` | string | Yes | `"Male"`, `"Female"`, or `"Other"` | `"Male"` |
| `preferredDoctorIds` | array of string | Yes | Doctor IDs the patient was registered for | `["abc123"]` |
| `doctorId` | string \| null | Yes | Currently assigned doctor. `null` until assigned | `"abc123"` |
| `tokenNumber` | number | Yes | Sequential token for the day (e.g. 1, 2, 3…) | `42` |
| `queueSequence` | number | Yes | Sort key for queue order. Walk-ins: same as `tokenNumber`. Fixed slots: `1000000 + minutesSinceMidnight` | `42` |
| `slotTime` | string \| null | No | ISO datetime for fixed-slot bookings. `null` for walk-ins | `"2026-07-10T10:30:00.000Z"` |
| `slotType` | string | Yes | `"walkin"` or `"fixed"` | `"walkin"` |
| `status` | string | Yes | See status flow below | `"waiting"` |
| `source` | string | Yes | `"reception"` or `"kiosk_qr_self_service"` | `"reception"` |
| `visitDate` | string | Yes | `YYYY-MM-DD` format | `"2026-07-10"` |
| `issuedAt` | string | Yes | ISO datetime when token was issued | `"2026-07-10T08:15:00.000Z"` |
| `consultStartAt` | string \| null | No | ISO datetime when doctor started consultation | `"2026-07-10T09:05:00.000Z"` |
| `consultEndAt` | string \| null | No | ISO datetime when consultation was completed | `"2026-07-10T09:13:00.000Z"` |
| `doctorNotes` | string \| null | No | Doctor's consultation notes | `"Prescribed paracetamol for fever"` |
| `vitals` | object \| null | No | See vitals structure below | see below |
| `fees` | object \| null | No | Legacy single-receipt fee record (see fees structure) | see below |
| `receipts` | array | No | Newer multi-receipt format. Each item is a receipt object | `[]` |
| `feeCollected` | boolean | Yes | `true` once any payment has been collected | `false` |
| `prescriptions` | array | No | Captured prescription images (see prescriptions structure) | `[]` |

### Visit Status Flow

```
waiting → called → in_consultation → completed
       ↘ no_show
       ↘ cancelled
```

| Status | Meaning |
|---|---|
| `waiting` | Token issued, patient in queue |
| `called` | Doctor called the patient (announced on board) |
| `in_consultation` | Doctor marked consultation as started |
| `completed` | Consultation done |
| `cancelled` | Visit cancelled |
| `no_show` | Patient did not respond when called |

### `vitals` Object Structure

```json
{
  "weight": 70,
  "temperature": 98.6,
  "bpSystolic": 120,
  "bpDiastolic": 80,
  "chiefComplaint": "Fever and cough for 3 days",
  "priorNotes": "Diabetic patient, on metformin"
}
```

### `fees` Object Structure (legacy)

```json
{
  "total": 350,
  "consultationFee": 300,
  "medicineFee": 50,
  "otherFee": 0,
  "mode": "cash",
  "collectedAt": "2026-07-10T09:20:00.000Z",
  "receiptNumber": "RCP-000042"
}
```

### `prescriptions` Array Item Structure

```json
{
  "id": "psc_1720601234567",
  "pageNumber": 1,
  "dataUrl": "data:image/jpeg;base64,/9j/4AAQ...",
  "capturedAt": "2026-07-10T09:12:00.000Z"
}
```

## Collection: `patients`

**Purpose:** Master patient directory. One document per patient across all visits.

**Document ID:** Auto-generated by Firestore.

**Relationships:** Referenced by `visits.patientId`.

### Fields

| Field | Type | Required | Description | Example |
|---|---|---|---|---|
| `name` | string | Yes | Patient's full name | `"Ravi Kumar"` |
| `mobile` | string | Yes | Mobile number (used to find returning patients) | `"9876543210"` |
| `age` | number | Yes | Age in years | `45` |
| `gender` | string | Yes | `"Male"`, `"Female"`, or `"Other"` | `"Male"` |
| `address` | string | No | Patient's address | `"12 Main Street, Chennai"` |
| `patientCode` | string | Yes | Unique patient code generated from `counters/patientId` | `"CLN-000001"` |
| `registrationSource` | string | Yes | `"reception"` or `"kiosk_qr_self_service"` | `"reception"` |
| `registeredAt` | string | Yes | ISO datetime of first registration | `"2026-07-10T08:15:00.000Z"` |
| `deletedAt` | string \| null | No | ISO datetime if soft-deleted. `null` for active patients | `null` |

> **Note:** Patients are soft-deleted (the `deletedAt` field is set) rather than permanently removed, so visit history remains intact.

## Collection: `counters`

**Purpose:** Atomic counters to generate sequential patient codes and daily token numbers.

**Document ID:** Two fixed IDs (see below). Never create other documents in this collection.

### Document 1: `patientId`

Increments by 1 each time a new patient is registered. Used to generate `CLN-000001`, `CLN-000002`, etc.

```json
{
  "value": 42
}
```

| Field | Type | Description |
|---|---|---|
| `value` | number | Current counter value. Next patient code = `CLN-` + zero-padded (`value + 1`) |

### Document 2: `tokens-YYYY-MM-DD`

One document per day, created automatically on the first token issued that day. Example document ID: `tokens-2026-07-10`.

```json
{
  "value": 15
}
```

| Field | Type | Description |
|---|---|---|
| `value` | number | Number of tokens issued today. Resets the next day (new document created) |

## Firestore Indexes

All indexes are on the `visits` collection. Create these in Firebase console → Firestore → Indexes → Add index.

| # | Collection | Fields | Query used for |
|---|---|---|---|
| 1 | `visits` | `doctorId` ASC, `status` ASC, `consultEndAt` DESC | Doctor's completed visits list |
| 2 | `visits` | `visitDate` ASC, `status` ASC | Daily queue filtered by status |
| 3 | `visits` | `visitDate` ASC, `tokenNumber` ASC | Queue board ordered by token |
| 4 | `visits` | `patientId` ASC, `issuedAt` DESC | Patient visit history |
| 5 | `visits` | `visitDate` ASC, `feeCollected` ASC | Pending fee collection list |

> **Note:** Firestore cannot auto-create composite indexes. Queries will fail with an error until the corresponding index exists and shows status **Enabled** (takes 1–5 minutes to build after creation).

## Security Rules Summary

| Collection | Unauth read | Unauth create | Unauth update/delete | Auth read | Auth write |
|---|---|---|---|---|---|
| `doctors` | ✅ | ❌ | ❌ | ✅ | ✅ |
| `visits` | ✅ | ✅ | ❌ | ✅ | ✅ |
| `clinicSettings` | ✅ | ❌ | ❌ | ✅ | ✅ |
| `counters` | ✅ | ✅ | ✅ | ✅ | ✅ |
| `patients` | ❌ | ❌ | ❌ | ✅ | ✅ |
| `users` | ❌ | ❌ | ❌ | ✅ | ✅ |
| `backups` | ❌ | ❌ | ❌ | ✅ | ✅ |

**Why `visits` allows public create:** The self-service kiosk (`self-service.html`) runs without login and must be able to submit a new visit document (token booking).

**Why `counters` allows full public write:** The token-number counter uses a Firestore transaction (`runTransaction`) which requires both read and write — needed from the self-service kiosk to generate sequential token numbers safely.

**Why `patients` is staff-only:** Patient records contain name, mobile, age, and address. Only authenticated staff can read or write this collection. The self-service kiosk stores patient details directly on the visit document instead; reception links or creates the patient record when processing the token.

Full rules are in [`firestore.rules`](../firestore.rules) in the repo root and in INSTALL.md § 6.
