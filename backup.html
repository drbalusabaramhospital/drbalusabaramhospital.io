Backup & Recovery

Download a full backup of all clinic data, or restore from a previous backup file.

📦 Download backup

Creates a JSON file containing all patients, visits, doctors, staff logins, clinic settings, and counters. Prescription images (base64) are included. Store the file in a safe location — Google Drive, email, or an external drive.

What IS backed up: all Firestore data — patients, visits, doctors, roles, settings, token counters, prescription images.
What is NOT backed up: Firebase Authentication credentials (email/password hashes). Passwords cannot be exported from Firebase by design — they are never exposed to client-side code. Staff login passwords are unaffected by restore operations on the same Firebase project. If migrating to a new project, staff accounts must be recreated manually in the Firebase Console.

🔄 Restore from backup

⚠ Warning: Restore overwrites existing documents with the same IDs. It does not delete documents that are not in the backup file. Always download a fresh backup before restoring.
